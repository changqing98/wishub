# 状态清单与提示文案

本文档用于统一问答页、知识库管理页、在线预览页的状态定义与推荐文案。

---

## 1. 文案原则

### 1.1 基本原则
- 简洁明确
- 优先说结论
- 不夸大系统能力
- 区分“不知道”和“请补充问题”
- 避免空泛报错

### 1.2 语气原则
- 面向外部客户，语气友好但克制
- 不拟人化过度
- 不做没有依据的承诺

---

## 2. 上传与知识库管理状态

| 状态 | 触发场景 | 推荐文案 |
|---|---|---|
| 未上传 | 初始空状态 | 还没有知识文件，上传后即可用于问答。 |
| 上传中 | 文件刚提交 | 正在上传文件，请稍候。 |
| 处理中 | 文件上传完成，正在解析 | 文件已上传，正在处理并纳入知识库。 |
| 上传成功 | 文件处理成功 | 文件已成功加入知识库。 |
| 部分成功 | 文件夹中部分文件失败 | 部分文件已成功导入，少量文件处理失败，请检查状态。 |
| 处理失败 | 文件无法处理 | 该文件暂时无法加入知识库，请重新上传或更换文件后重试。 |
| 不支持类型 | 文件类型不在支持范围内 | 当前仅支持 Markdown、TXT、PDF、Word 文件。 |
| 替换成功 | 更新文件成功 | 文件已更新，最新内容将用于后续问答。 |
| 删除成功 | 删除文件成功 | 文件已从知识库中移除。 |
| 列表加载失败 | 管理页读取异常 | 文件列表暂时加载失败，请稍后重试。 |

---

## 3. 问答状态

| 状态 | 触发场景 | 推荐文案 |
|---|---|---|
| 未提问 | 问答页初始状态 | 请输入你想了解的问题，我会优先根据知识库回答。 |
| 提问中 | 用户刚提交问题 | 正在接收你的问题。 |
| 检索中 | 系统正在查找知识依据 | 正在知识库中查找相关内容，请稍候。 |
| 回答成功 | 找到足够依据 | 已根据知识库找到相关答案。 |
| 无依据 | 没有找到充分支持信息 | 当前知识库中没有足够信息回答这个问题。 |
| 问题不清晰 | 问题语义不足 | 为了更准确地回答你，请补充一下具体产品、功能、页面或场景。 |
| 系统异常 | 服务异常 | 当前暂时无法完成回答，请稍后重试。 |

---

## 4. “不知道”型回答文案

以下文案用于知识库无明确依据时。

### 标准版
- 当前知识库中没有足够信息回答这个问题。

### 稍微友好版
- 我暂时未在当前知识库中找到支持这个问题的明确依据。

### 带建议版
- 当前知识库中没有足够信息回答这个问题。你也可以换一种更具体的问法试试。

### 使用规则
- 用于“问题明确，但知识库无依据”场景
- 不应在这类文案后追加知识库外猜测

---

## 5. “请补充问题”型文案

以下文案用于问题不够清晰时。

### 通用版
- 为了更准确地回答你，请补充一下具体产品、功能、页面或使用场景。

### 偏操作问题
- 你可以补充一下你要操作的功能、页面名称，或遇到的具体步骤。

### 偏技术问题
- 请补充一下相关模块、接口名称、错误码或报错信息，我再继续帮你判断。

### 偏业务规则问题
- 你可以说明一下具体业务场景，例如套餐、权限、退款、发票等，我再进一步回答。

### 使用规则
- 用于“问题模糊、信息不足、指代不明确”场景
- 与“不知道”文案要严格区分

---

## 6. 回答成功文案结构

有效回答建议统一采用如下结构：

### 模板
- 答案：<直接结论>
- 说明：<一句补充说明>
- 参考依据：<文档名 + 片段 + 预览入口>

### 示例
- 答案：你可以在“开发者设置”页面配置回调地址。
- 说明：根据当前知识库，回调地址需要使用可访问的 HTTPS 地址。
- 参考依据：《开发者接入指南》 / “回调地址必须为 HTTPS，并且可以被平台访问……” / 查看引用位置

## 10. 空状态文案

## 10.1 问答页空状态
- 请输入你想了解的问题，我会优先根据知识库回答。
- 例如：如何开通企业版？退款规则是什么？API 如何鉴权？

## 10.2 知识库管理页空状态
- 还没有知识文件，上传后即可用于问答。
- 支持上传 Markdown、TXT、PDF、Word 文件，以及保留目录结构的文件夹。

## 10.3 证据区空状态
- 当前回答暂无可展示的引用依据。

注意：首版正式回答若无证据，一般应归入“不知道”场景，而不是正常成功回答。

## 11. 错误状态文案

| 场景 | 推荐文案 |
|---|---|
| 上传失败 | 文件上传失败，请稍后重试。 |
| 解析失败 | 该文件暂时无法处理，请检查文件内容后重新上传。 |
| 提问失败 | 当前暂时无法处理你的问题，请稍后再试。 |
| 预览失败 | 当前暂时无法打开引用内容，请稍后再试。 |
| 历史加载失败 | 最近提问记录暂时无法加载。 |

---

## 12. 文案禁忌

以下表达建议避免：
- “我猜测……”
- “大概是……”
- “应该没问题……”
- “虽然没有查到，但通常来说……”
- “可能是这个意思……”

原因：这些表达会直接削弱产品的可信度边界。

---

## 13. 测试重点提示

测试时应重点校验以下文案是否被正确区分：

1. **有依据回答**
   - 必须出现证据卡片
2. **无依据拒答**
   - 不能追加猜测性答案
3. **问题不清晰**
   - 必须出现澄清提示，而不是错误返回
4. **在线预览**
   - 必须明确提示仅展示相关片段
5. **管理页失败状态**
   - 不能只显示“失败”，至少要能区分上传失败/处理失败
