## 2. 产品概述

### 2.1 产品定义
知识库智能客服是 `wishub（智慧中枢）` mvp 版本中的核心能力之一，面向 **外部客户** 提供 **Web 页面智能问答服务**。

它的核心能力不是开放式聊天，而是：
- 基于企业上传的知识库进行问答
- 所有有效回答都要可追溯、可核验
- 无证据时不乱答
- 问题不明确时先引导客户补充

### 2.2 产品定位
这是一个 **严格证据驱动型知识问答系统**，而不是通用聊天机器人。

### 2.3 MVP 目标
首版重点验证以下价值：
1. 企业可把知识资料以文件/文件夹形式导入系统
2. 客户可在 Web 页面通过自然语言提问
3. 系统可基于知识库返回贴题答案
4. 每次有效回答都带有证据
5. 没有依据时系统能稳定输出“不知道”或引导补充

## 3. 已确认需求与产品约束

### 3.1 用户与产品形态
- 目标用户：外部客户
- 产品形态：Web 页面

### 3.2 文件类型支持
首版支持：
- Markdown
- TXT
- PDF
- Word

### 3.3 知识组织方式
- 支持文件夹上传
- 保留原始目录结构
- 管理方式接近代码仓库 / 文档仓库

### 3.4 问答策略
- 仅基于知识库回答
- 无知识库依据时，不允许补充知识库外结论
- 回答策略采用严格模式

### 3.5 证据展示
每次有效回答必须展示：
- 文档名
- 命中片段
- 在线预览跳转入口

### 3.6 原文预览规则
- 跳转目标：在线预览页或预览弹层
- 支持定位到对应页码或段落
- 客户只能查看引用片段相关内容
- 不开放完整原文浏览

### 3.7 模糊问题处理策略
- 采用友好型澄清策略
- 问题不清楚时，不直接乱答
- 优先提示客户补充产品、功能、场景、报错等关键信息

### 3.8 会话能力
- 保留当前用户最近提问记录
- 首版不要求复杂历史会话管理

### 3.9 首版不做
- 权限控制
- 多租户
- 知识库外自由补答
- 客服转人工
- 高级运营分析报表

## 5. 产品目标与非目标

### 5.1 产品目标
1. 让客户能在 Web 页面直接提问
2. 让系统仅在知识库范围内作答
3. 让客户可快速验证答案出处
4. 让系统在不确定时保持克制
5. 让企业能以目录树形式维护知识库

### 5.2 非目标
首版不追求：
- 通用聊天体验
- 知识库外常识补全
- 高度拟人化客服人格
- 多轮复杂推理对话
- 完整全文开放阅读
- 复杂后台权限和组织管理

---

## 6. 产品原则

### 原则 1：可信优先，宁可少答，不可乱答
系统价值来自“可信”，而不是“看起来什么都能回答”。

### 原则 2：知识库是唯一事实边界
首版中，所有答案都必须落在知识库事实边界内。

### 原则 3：证据先于表达
没有证据支撑的答案，不视为有效答案。

### 原则 4：先理解问题，再组织答案
不能只抓关键词返回相关内容，必须真正回应用户问题。

### 原则 5：模糊问题先澄清，不做无依据猜测
问题不够清晰时，应优先引导补充，而不是猜测性作答。

### 原则 6：证据可核验，但原文受控
用户可以验证引用位置，但不能无限制阅读全文。

## 7. 功能范围与模块拆解

## 7.1 模块 A：知识库上传与导入

### 功能目标
支持企业将资料导入系统，并纳入问答范围。

### 范围
- 单文件上传
- 多文件上传
- 文件夹上传
- 保留目录结构

### 产品要求
- 上传成功后，文件可进入知识库范围
- 上传中/处理中/失败状态需可见
- 文件夹层级不能丢失
- 不支持文件需明确提示

### 子功能拆解
- A1. 单文件上传
- A2. 文件夹上传
- A3. 上传任务状态展示
- A4. 导入结果反馈

---

## 7.2 模块 B：知识库管理

### 功能目标
让运营方像管理文档仓库一样维护知识内容。

### 核心能力
- 目录树浏览
- 文件列表查看
- 查看文件状态
- 删除文件
- 替换文件
- 按路径理解文件归属

### 产品要求
- 路径结构应清晰可见
- 文件路径可以作为来源上下文的一部分
- 更新动作应尽量简洁明确

### 子功能拆解
- B1. 目录树管理
- B2. 文件状态展示
- B3. 文件删除
- B4. 文件替换

---

## 7.3 模块 C：客户问答

### 功能目标
为外部客户提供贴题、可信、克制的问答体验。

### 回答规则
1. 系统只在知识库内寻找依据
2. 若存在足够依据，则输出答案
3. 若无依据，则回答“不知道”或等价表述
4. 若问题不明确，则先引导客户补充
5. 不允许为追求完整而生成无证据结论

### 回答结构建议
每次有效回答包含三部分：
- 直接答案
- 简要说明
- 参考依据

### 子功能拆解
- C1. 问题输入
- C2. 问题校验与澄清
- C3. 基于知识库的回答
- C4. 无依据拒答
- C5. 最近提问记录

---

## 7.4 模块 D：证据展示与在线预览

### 功能目标
让客户验证答案可信度，同时控制原文暴露范围。

### 证据卡片内容
- 文档名
- 文档路径
- 命中片段
- 在线预览入口

### 在线预览规则
- 跳转至在线预览页或预览弹层
- 优先定位到相关页码或段落
- 高亮相关片段
- 仅展示引用片段及有限上下文
- 不支持完整原文通读

### 子功能拆解
- D1. 证据卡片
- D2. 多证据展示
- D3. 在线预览与页码定位

## 9. 信息架构与页面结构

首版建议包含 3 个核心页面 / 区域：
1. 客户问答页
2. 在线预览页 / 预览弹层
3. 知识库管理页

---

## 9.1 页面一：客户问答页

### 页面目标
- 让外部客户快速提问
- 让客户清楚看到答案
- 让客户快速理解答案依据
- 让客户在问题模糊时获得友好提示
- 让客户能回看最近提问记录

### 页面布局建议
```text
┌──────────────────────────────────────────────┐
│ 顶部说明区：产品说明 / 使用提示               │
├──────────────────────────────────────────────┤
│ 左侧：最近提问记录   │ 右侧：问答主区域        │
│ - 最近 5~10 条问题   │ ┌──────────────────────┐ │
│ - 点击可复用         │ │ 对话记录区           │ │
│                      │ │ - 用户问题           │ │
│                      │ │ - 系统回答           │ │
│                      │ │ - 证据卡片           │ │
│                      │ └──────────────────────┘ │
│                      │ 输入框 + 提问按钮        │
└──────────────────────────────────────────────┘
```

### 页面模块说明
- 顶部说明区：提前建立产品边界认知
- 最近提问记录：帮助当前用户快速回看和复用问题
- 问答对话区：承载用户问题、系统回答、状态反馈、证据卡片
- 输入区：输入框 + 提问按钮 + 占位提示文案

### 关键交互规则
#### 场景 A：问题明确且有依据
系统返回：
- 直接答案
- 简要解释
- 证据卡片

#### 场景 B：问题明确但无依据
系统返回：
- 不知道 / 当前知识库暂无足够信息
- 不给出知识库外补充结论

#### 场景 C：问题不明确
系统返回：
- 友好型澄清提示
- 引导用户补充产品、模块、页面、报错、场景等信息

#### 场景 D：用户点击证据卡片
系统动作：
- 打开在线预览页或预览弹层
- 自动定位到相关页码/片段
- 高亮引用区域

## 9.2 页面二：在线预览页 / 预览弹层

### 页面目标
- 帮助客户核验答案出处
- 支持页码定位和片段高亮
- 保持原文受控，不开放全文浏览

### 页面定位
这是“证据核验页面”，不是完整文档阅读器。

### 页面布局建议
```text
┌──────────────────────────────────────────────┐
│ 文档名 / 路径 / 页码信息                      │
├──────────────────────────────────────────────┤
│ 片段预览区                                    │
│ - 高亮命中内容                                │
│ - 展示有限上下文                              │
│ - 提示：仅展示与当前回答相关内容              │
├──────────────────────────────────────────────┤
│ 上一处命中 / 下一处命中 / 关闭                 │
└──────────────────────────────────────────────┘
```

### 页面模块说明
- 文档信息区：文档名、路径、页码、文件类型
- 片段预览区：命中片段、上下文、高亮标记
- 命中切换区：用于多个命中位置切换

### 展示边界
- 客户只能查看片段及有限上下文
- 不提供“阅读全文”或“导出全文”能力

## 10. 页面流转关系

```text
客户问答页
  ├─ 提问成功 → 回答成功卡 + 证据卡片
  │                   └─ 点击证据 → 在线预览层
  ├─ 提问成功 → 拒答卡
  └─ 提问成功 → 澄清提示卡

知识库管理页
  ├─ 上传文件 → 文件状态变化
  ├─ 上传文件夹 → 目录树更新
  ├─ 删除文件 → 文件移除
  └─ 替换文件 → 文件状态刷新
```

## 11. 状态定义与推荐文案

## 11.1 上传与知识库管理状态
| 状态 | 触发场景 | 推荐文案 |
|---|---|---|
| 未上传 | 初始空状态 | 还没有知识文件，上传后即可用于问答。 |
| 上传中 | 文件刚提交 | 正在上传文件，请稍候。 |
| 处理中 | 文件上传完成，正在解析 | 文件已上传，正在处理并纳入知识库。 |
| 上传成功 | 文件处理成功 | 文件已成功加入知识库。 |
| 部分成功 | 文件夹中部分文件失败 | 部分文件已成功导入，少量文件处理失败，请检查状态。 |
| 处理失败 | 文件无法处理 | 该文件暂时无法加入知识库，请重新上传或更换文件后重试。 |
| 不支持类型 | 文件类型不在支持范围内 | 当前仅支持 Markdown、TXT、PDF、Word 文件。 |
| 替换成功 | 更新文件成功 | 文件已更新，最新内容将用于后续问答。 |
| 删除成功 | 删除文件成功 | 文件已从知识库中移除。 |
| 列表加载失败 | 管理页读取异常 | 文件列表暂时加载失败，请稍后重试。 |

## 11.2 问答状态
| 状态 | 触发场景 | 推荐文案 |
|---|---|---|
| 未提问 | 问答页初始状态 | 请输入你想了解的问题，我会优先根据知识库回答。 |
| 提问中 | 用户刚提交问题 | 正在接收你的问题。 |
| 检索中 | 系统正在查找知识依据 | 正在知识库中查找相关内容，请稍候。 |
| 回答成功 | 找到足够依据 | 已根据知识库找到相关答案。 |
| 无依据 | 没有找到充分支持信息 | 当前知识库中没有足够信息回答这个问题。 |
| 问题不清晰 | 问题语义不足 | 为了更准确地回答你，请补充一下具体产品、功能、页面或场景。 |
| 系统异常 | 服务异常 | 当前暂时无法完成回答，请稍后重试。 |

## 11.3 “不知道”型回答文案
- 当前知识库中没有足够信息回答这个问题。
- 我暂时未在当前知识库中找到支持这个问题的明确依据。
- 当前知识库中没有足够信息回答这个问题。你也可以换一种更具体的问法试试。

使用规则：
- 用于“问题明确，但知识库无依据”场景
- 不应在这类文案后追加知识库外猜测

## 11.4 “请补充问题”型文案
- 为了更准确地回答你，请补充一下具体产品、功能、页面或使用场景。
- 你可以补充一下你要操作的功能、页面名称，或遇到的具体步骤。
- 请补充一下相关模块、接口名称、错误码或报错信息，我再继续帮你判断。

使用规则：
- 用于“问题模糊、信息不足、指代不明确”场景
- 与“不知道”文案要严格区分

## 11.5 证据卡片文案
建议字段：
- 文档名
- 文档路径
- 引用片段
- 查看引用位置

辅助说明文案：
- 以下内容来自知识库中的相关文档
- 你可以查看引用位置来核验答案依据
- 仅展示与当前回答相关的引用内容

## 12. 线框图说明与评审重点

## 12.1 问答页 Review 重点
- 是否真正突出“答案 + 证据”双核心结构
- 最近提问记录是否为辅助区域，而非主操作区
- “不知道”和“请补充问题”是否可明显区分
- 输入区是否始终明确可用

## 12.2 预览页 Review 重点
- 用户一进入是否能立刻看到命中内容
- 页码或段落定位是否明确
- 是否存在可继续浏览整篇文档的路径
- “证据核验页”是否被误做成“文档阅读页”

## 12.3 管理页 Review 重点
- 上传按钮是否足够明显
- 目录树是否足够清晰
- 文件状态是否一眼可分辨
- 删除/替换操作是否低学习成本
- 部分成功场景是否会误导用户

## 13.5 关键接口约束
### 提交问题 `POST /api/mvp/chat/questions`
返回结果必须明确区分：
- `ANSWERED`
- `NEED_CLARIFICATION`
- `NO_ANSWER`
- `ERROR`

合同约束：
- `ANSWERED` 时，`answerText` 必填，`evidences.length >= 1`
- `NEED_CLARIFICATION` 时，`clarificationPrompts.length >= 1`
- `NO_ANSWER` 时，不应返回证据列表冒充有效答案
- `ERROR` 时，前端应展示系统异常状态，而非业务拒答状态

### 获取证据预览 `GET /api/mvp/evidences/{evidenceId}/preview`
合同约束：
- 不返回完整文档正文
- 不返回可拼接成全文的长列表内容
- `previewMode` 在 mvp 中固定为 `SNIPPET_ONLY`
- 如果不可预览，应返回明确错误码

### 上传与文件管理
- 服务端必须保留文件夹 `relativePath`
- 替换后保留原路径
- 删除成功后，该文件不得继续参与问答
- `READY` 以外状态文件不得参与问答

## 13.6 错误码建议
| 错误码 | 场景 | 前端处理建议 |
|---|---|---|
| VALIDATION_ERROR | 参数错误 | 展示表单级提示 |
| QUESTION_TOO_SHORT | 问题过短 | 可转为澄清提示 |
| UNSUPPORTED_FILE_TYPE | 文件类型不支持 | 展示支持范围 |
| FILE_PROCESSING_FAILED | 文件处理失败 | 展示失败状态与重试建议 |
| FILE_NOT_FOUND | 文件不存在 | 刷新列表并提示 |
| PREVIEW_NOT_AVAILABLE | 预览不可用 | 展示预览失败提示 |
| CHAT_SERVICE_UNAVAILABLE | 问答服务异常 | 展示系统异常状态 |
| TREE_LOAD_FAILED | 目录树加载失败 | 展示加载失败状态 |

## 14. 测试验收清单

## 14.1 验收原则
- 有依据才回答
- 无依据不乱答
- 问题模糊先澄清
- 证据必须能支撑答案
- 客户只能查看引用片段，不能阅读全文

### 一票否决项
以下任一问题出现，应视为验收不通过：
1. 没有知识库依据却输出确定性答案
2. 回答明显偏题或只返回泛泛相关内容
3. 证据片段不能支撑答案核心结论
4. 外部客户可通过预览查看完整原文
5. 已失败或已删除文件仍参与问答

## 14.2 测试范围
覆盖以下模块：
- 客户问答
- 证据展示与在线预览
- 知识库上传与管理
- 最近提问记录
- 页面状态与提示文案

## 14.3 模块级验收清单

### A. 客户问答
#### 问题输入
- 用户输入正常问题并提交，系统成功接收问题
- 用户点击提问按钮 / 回车提交，系统开始问答流程
- 长问题可正常展示，不截断关键内容

#### 有依据回答
- 知识库存在明确答案时，系统返回贴题答案
- 回答成功时，展示答案、说明、参考依据
- 单篇文档支持时，显示对应证据卡片
- 多篇文档共同支持时，展示多条证据且结构清晰

#### 无依据拒答
- 知识库中没有相关内容时，系统返回“不知道”型回答
- 有相关内容但证据不足时，系统拒答
- 问题超出知识库范围时，系统拒答且不自由发挥

#### 模糊问题澄清
- 问题过短或上下文不足时，系统提示补充信息
- 指代不明时，系统先澄清，不直接作答
- 用户补充后，系统能继续进入正常问答流程

### B. 证据展示与在线预览
#### 证据卡片展示
- 回答成功后，页面出现证据卡片
- 至少包含文档名、路径/片段、查看入口
- 多条证据展示顺序清晰，不错误拼接

#### 在线预览
- PDF 预览可定位到对应页码
- Word 预览可定位到对应页码或等价位置
- Markdown/TXT 预览可定位到对应段落
- 命中片段高亮清晰可辨
- 多命中可切换查看

#### 原文边界控制
- 外部客户打开预览时，仅看到引用片段及有限上下文
- 不存在阅读全文入口
- 页面明确提示仅展示相关引用内容

### C. 知识库上传与管理
#### 单文件上传
- 支持 Markdown、TXT、PDF、Word 上传
- 不支持类型时，文案明确说明支持范围

#### 文件夹上传与目录结构
- 上传多层级文件夹时，目录结构被保留
- 支持文件被识别并处理
- 文件夹部分失败时，成功与失败文件清晰区分

#### 文件状态、删除与替换
- 处理中状态不会误显示为可用
- 失败文件易识别
- 删除后，后续问答不再引用该文件
- 替换后，问答引用更新为新内容

### D. 最近提问记录
- 首次进入问答页时，显示无记录空状态
- 提问成功后，最近提问中出现该问题
- 点击历史问题可复用或展开
- 最近记录加载失败不影响主问答功能

### E. 状态与文案
- 有依据回答、不知道、澄清提示、系统异常四类状态区分清晰
- 拒答文案中不出现猜测性表达
- 澄清文案足够具体，可指导用户补充信息
- 预览提示文案明确原文边界

## 14.4 端到端验收场景

### 场景 1：标准成功问答
1. 上传一组正常知识文件
2. 用户提出一个知识库中有明确答案的问题
3. 系统返回贴题答案
4. 系统展示证据卡片
5. 用户点击查看引用位置
6. 进入在线预览并定位到对应页码/片段

通过标准：
- 全链路顺畅
- 答案正确
- 证据可核验
- 无多余暴露

### 场景 2：无依据拒答
1. 用户提出一个知识库中不存在答案的问题
2. 系统返回“不知道”型文案
3. 系统不附带知识库外猜测答案

### 场景 3：模糊问题澄清
1. 用户输入模糊问题
2. 系统提示用户补充问题
3. 用户补充后再次提问
4. 系统正常回答或拒答

### 场景 4：文件夹导入与目录树生效
1. 上传一个多层级文件夹
2. 系统保留目录结构
3. 页面正确展示目录树
4. 从其中一个文档中提问并命中答案

### 场景 5：原文边界控制
1. 用户点击预览证据
2. 系统展示对应页码和片段
3. 用户尝试继续阅读更多原文
4. 系统不提供全文阅读能力

## 14.5 缺陷优先级建议
### P0
- 乱答
- 答非所问
- 错引证据
- 暴露全文
- 已删除/失败文件继续生效

### P1
- 证据排序不理想
- 最近提问记录异常
- 目录树局部展示问题
- 预览定位不够精准但仍可用

### P2
- 样式不统一
- 辅助文案优化空间
- 轻微布局错位

## 16. 风险与依赖提示

### 16.1 高依赖点
- PDF / Word 在线预览及页码定位体验
- 文件夹上传后目录结构保留的一致性
- 问题模糊与无依据场景的边界区分
- 证据片段与答案的一致性控制

### 16.2 高风险点
- 回答命中资料但没有真正回答问题
- 证据片段与答案存在弱关联
- 预览页虽限制全文，但实际仍可暴露过多内容
- 文件已失败却被误认为已生效

## 17. MVP 成功判断

若首版上线后满足以下条件，可视为 MVP 成立：
- 用户提出常见问题时，系统大多数情况下能返回证据型回答
- 无依据问题不会出现明显乱答
- 模糊问题能通过友好澄清提升问答成功率
- 客户能理解证据卡片并愿意点击验证
- 运营方能稳定维护知识内容

---

## 18. 上线前最小通过标准

上线前建议至少满足以下条件：
1. 核心问答链路可稳定跑通
2. 有依据问题能正确回答并带证据
3. 无依据问题不会乱答
4. 模糊问题会引导补充
5. 预览可定位片段但不能阅读全文
6. 文件夹结构可保留并正确生效
7. 删除/替换后问答结果会同步更新

若以上任一关键项未达标，不建议以 MVP 正式对外开放。
