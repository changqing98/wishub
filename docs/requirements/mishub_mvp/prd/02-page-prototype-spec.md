# Web 首版页面原型说明

本文档用于指导设计和研发完成知识库智能客服 MVP 的页面结构与交互定义。

---

## 1. 页面清单

首版建议包含 3 个核心页面 / 区域：

1. 客户问答页
2. 在线预览页 / 预览弹层
3. 知识库管理页

---

## 2. 页面一：客户问答页

## 2.1 页面目标
- 让外部客户快速提问
- 让客户清楚看到答案
- 让客户快速理解答案依据
- 让客户在问题模糊时获得友好提示
- 让客户能回看最近提问记录

## 2.2 页面布局建议

```text
┌──────────────────────────────────────────────┐
│ 顶部说明区：产品说明 / 使用提示               │
├──────────────────────────────────────────────┤
│ 左侧：最近提问记录   │ 右侧：问答主区域        │
│ - 最近 5~10 条问题   │ ┌──────────────────────┐ │
│ - 点击可复用         │ │ 对话记录区           │ │
│                      │ │ - 用户问题           │ │
│                      │ │ - 系统回答           │ │
│                      │ │ - 证据卡片           │ │
│                      │ └──────────────────────┘ │
│                      │ 输入框 + 提问按钮        │
└──────────────────────────────────────────────┘
```

## 2.3 页面模块说明

### 模块 A：顶部说明区
目的：提前建立产品边界认知。

建议内容：
- 系统会优先依据知识库回答问题
- 回答附带引用依据
- 问题不清楚时会提示补充
- 无足够依据时会明确说明不知道

### 模块 B：最近提问记录
目的：帮助当前用户快速回看和复用问题。

展示建议：
- 默认展示最近 5~10 条
- 文案只展示问题摘要
- 点击后可直接回填输入框，或直接展开对应问答

交互要求：
- 当前用户维度展示
- 问题过长时单行省略
- 允许“重新提问”

### 模块 C：问答对话区
目的：承载用户问题、系统回答、状态反馈、证据卡片。

内容顺序建议：
1. 用户问题
2. 系统状态（检索中 / 澄清中 / 不知道 / 回答成功）
3. 系统回答正文
4. 证据卡片

### 模块 D：输入区
建议包含：
- 输入框
- 提问按钮
- 占位提示文案

占位文案示例：
- 请输入你想了解的问题，例如：如何开通企业版？
- 请输入更具体的问题，我会优先根据知识库回答。

---

## 2.4 问答页关键交互规则

### 场景 A：问题明确且有依据
系统返回：
- 直接答案
- 简要解释
- 证据卡片

### 场景 B：问题明确但无依据
系统返回：
- 不知道 / 当前知识库暂无足够信息
- 不给出知识库外补充结论

### 场景 C：问题不明确
系统返回：
- 友好型澄清提示
- 引导用户补充产品、模块、页面、报错、场景等信息

### 场景 D：用户点击证据卡片
系统动作：
- 打开在线预览页或预览弹层
- 自动定位到相关页码/片段
- 高亮引用区域

---

## 2.5 推荐回答区结构

```text
用户：如何配置回调地址？

系统：
答案：你可以在开发者设置页面中填写回调地址，并完成验证后启用。
说明：根据当前知识库，回调地址需要使用可访问的 HTTPS 地址。
参考依据：
- 《开发者接入指南》
  “回调地址必须为 HTTPS，并且可以被平台访问……”
  [查看引用位置]
```

## 3. 页面二：在线预览页 / 预览弹层

## 3.1 页面目标
- 帮助客户核验答案出处
- 支持页码定位和片段高亮
- 保持原文受控，不开放全文浏览

## 3.2 页面定位
这是“证据核验页面”，不是完整文档阅读器。

## 3.3 页面布局建议

```text
┌──────────────────────────────────────────────┐
│ 文档名 / 路径 / 页码信息                      │
├──────────────────────────────────────────────┤
│ 片段预览区                                    │
│ - 高亮命中内容                                │
│ - 展示有限上下文                              │
│ - 提示：仅展示与当前回答相关内容              │
├──────────────────────────────────────────────┤
│ 上一处命中 / 下一处命中 / 关闭                 │
└──────────────────────────────────────────────┘
```

## 3.4 页面模块说明

### 模块 A：文档信息区
展示：
- 文档名
- 所属目录路径
- 命中页码
- 文档类型

### 模块 B：片段预览区
展示：
- 命中片段
- 片段上下文
- 高亮标记

控制要求：
- 不能提供整篇连续滚动阅读能力
- 不能从预览无限扩展到完整全文
- 需要明确提示“仅展示引用相关内容”

### 模块 C：命中切换区
如果一个回答关联多个命中位置：
- 支持查看上一处 / 下一处
- 同一文档内多个命中点可切换

---

## 3.5 在线预览交互规则

### PDF / Word
- 进入后优先定位到相关页码
- 页码信息需明确可见
- 命中内容需高亮

### Markdown / TXT
- 进入后优先定位到相关段落
- 展示文件路径和相邻上下文

### 展示边界
- 客户只能查看片段及有限上下文
- 不提供“阅读全文”或“导出全文”能力

## 4. 页面三：知识库管理页

## 4.1 页面目标
- 让运营方维护知识库资料
- 让目录结构清晰可管理
- 让文件状态、操作结果明确可见

## 4.2 页面布局建议

```text
┌──────────────────────────────────────────────┐
│ 顶部操作区：上传文件 / 上传文件夹             │
├──────────────────────────────────────────────┤
│ 左侧：目录树          │ 右侧：文件列表/详情     │
│ - 展开/折叠           │ - 文件名                │
│ - 层级路径            │ - 类型                  │
│ - 当前选中节点        │ - 状态                  │
│                      │ - 更新时间              │
│                      │ - 删除 / 替换           │
└──────────────────────────────────────────────┘
```

## 4.3 页面模块说明

### 模块 A：顶部操作区
建议按钮：
- 上传文件
- 上传文件夹
- 刷新状态

### 模块 B：目录树
要求：
- 支持展开/折叠
- 保留原始层级结构
- 节点命名清晰
- 当前目录高亮

### 模块 C：文件列表 / 详情区
建议字段：
- 文件名
- 文件类型
- 所属路径
- 上传/处理状态
- 最近更新时间
- 操作项（删除 / 替换）

---

## 4.4 知识库管理关键交互

### 场景 A：上传单文件
- 点击上传文件
- 选择文件
- 开始上传
- 展示上传状态
- 处理完成后进入目录树

### 场景 B：上传文件夹
- 点击上传文件夹
- 选择本地文件夹
- 系统按原始结构导入
- 前端展示目录树结果

### 场景 C：文件处理失败
- 文件状态标记为失败
- 允许删除或重新替换
- 错误提示应尽量可理解

### 场景 D：替换文件
- 保留原路径
- 新文件覆盖旧内容
- 更新状态和更新时间

---

## 5. 关键流程说明

## 5.1 流程一：客户提问并获得答案
1. 用户输入问题
2. 系统进入检索中状态
3. 系统命中知识片段
4. 输出答案与证据卡片
5. 用户点击查看引用位置
6. 打开在线预览并定位页码

## 5.2 流程二：客户提问但知识库无依据
1. 用户输入问题
2. 系统检索后无明确依据
3. 输出“不知道”型回答
4. 不追加知识库外推断

## 5.3 流程三：客户问题不清晰
1. 用户输入模糊问题
2. 系统识别问题上下文不足
3. 输出澄清提示
4. 用户补充后重新提问

## 5.4 流程四：运营方上传文件夹
1. 进入知识库管理页
2. 选择上传文件夹
3. 系统解析文件树
4. 展示目录结构和处理状态
5. 成功文件进入问答范围

---

## 6. 设计注意事项

### 6.1 问答页重点
- 回答区域不要过度花哨
- 证据卡片应足够醒目
- “不知道”和“请补充问题”文案要明显区分

### 6.2 预览页重点
- 突出命中位置
- 明确页码
- 强调只展示相关内容

### 6.3 管理页重点
- 目录树体验要稳定
- 文件状态要一眼可辨
- 上传与替换动作要低学习成本
